Difficulty: Medium Category: Reconnaissance Platform: Hack The Box Date: October 2025

This detailed writeup documents a multi-stage CTF challenge involving initial credential usage on an FTP server, SSH key extraction, NFS enumeration, and SNMP exploitation to locate multiple flags and target credentials across several virtual machines.

📋 Overview

This writeup documents a hands-on CTF exercise starting with limited credentials. The objective was to enumerate multiple target systems, uncover hidden files/credentials, locate the flag.txt file, and identify the password for the user HTB. The exercise required a combination of network scanning, service enumeration, and exploitation techniques.

🔍 Reconnaissance

Initial network scanning identified active services on Target 1 (10.129.249.89). Tested provided credentials (username: ceil, password: [REDACTED]) against open FTP services.

nmap -sV 10.129.249.89
Key Finding: Four primary ports were open (21, 22, 53, 2121), and the given credentials were valid for two ProFTPD instances (Ports 21 and 2121).

🔎 Enumeration

Enumerated FTP directory structure to identify potential hidden files. Later, pivoted to Target 2 (10.129.220.178) for NFS shares and Target 3 (10.129.250.100) for SNMP.

ls -la
Critical Discovery 1: A hidden folder named .ssh was present on the FTP server. Found and downloaded the private key file: id_rsa.
showmount -e 10.129.220.178
Critical Discovery 2: The shared folder /TechSupport was exposed. A file inside (ticket4238791283782.txt) contained RDP credentials for user alex ([REDACTED]).
snmpwalk -v 2c -c public 10.129.250.100
Critical Discovery 3: SNMP enumeration extracted credentials admin:[REDACTED], leading to mail service access.

⚡ Exploitation

Exploitation involved utilizing discovered credentials and keys to establish footholds across the targets.

⚠️ Important: The NFS share was mounted locally to retrieve credentials before attempting the RDP connection.
ssh -i id_rsa ceil@10.129.249.89
xfreerdp /v:10.129.220.178 /u:alex /p:'[REDACTED]'
imaps -u admin -p [REDACTED]

🎯 Post-Exploitation

Establishing shells and sessions allowed for final objective retrieval across all targets.

find / -name flag.txt 2>/dev/null
✅ Success: Flag found at /home/ceil/flag.txt ([REDACTED]). HTB user password successfully retrieved via RDP SQL access and IMAPS file extraction.

📚 Key Learnings

  • Thorough Network Scanning: Vital for finding non-standard ports (like Port 2121) and UDP services (for SNMP).
  • Credential Reuse: Always test provided credentials on all services (FTP, SSH, RDP).
  • Hidden Directories: Enumerating hidden files (e.g., .ssh) or NFS shares is often the key to the next step.
  • Weak Protocols: NFS and SNMP are high-value targets for credential extraction.
  • Adaptability: Different targets require varying enumeration tools (Nmap, showmount, snmpwalk).

🛠️ Tools Used

  • Nmap: Used for comprehensive network scanning and service version detection.
  • FTP Client: Used to access the ProFTPD services and download the private key.
  • SSH Client: Used with the id_rsa key for remote shell access.
  • showmount & mount: Used to enumerate and access the exported NFS shares.
  • xfreerdp: Used to establish a Remote Desktop Protocol connection to the Windows server.
  • snmpwalk: Used to query the SNMP agent and dump sensitive configuration data (credentials).
  • imaps client: Used to access the mail server to retrieve the final password file.
