Information Gathering (Web Edition)

Difficulty: Easy Category: OSINT & Web Enumeration Platform: HTB Academy Date: October 2025

A detailed, step-by-step guide to mastering web reconnaissance, including WHOIS, subdomain enumeration, directory brute-forcing, API key discovery, and email harvesting using Gobuster, FFUF, and ReconSpider.

📋 Overview

This module solidifies understanding of web reconnaissance techniques. The journey encompasses subdomain enumeration, directory brute-forcing, API key discovery, and email harvesting. This guide functions as an actionable checklist for mapping a target's digital footprint.

🔍 Reconnaissance

Initial domain-level OSINT focuses on understanding domain ownership and registrar details to establish the target's fundamental footprint.

whois inlanefreight.com
Key Finding: The IANA ID was 468, corresponding to Namecheap. IANA IDs are crucial for tracking domain ownership and potential vulnerabilities in domain management systems.

🔎 Enumeration

Subdomains often hide critical assets like admin panels or misconfigured services. Local DNS resolution was configured via the /etc/hosts file before running Gobuster and FFUF.

gobuster vhost -u http://inlanefreight.htb:44038 -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-110000.txt --append-domain -t 50
Key Finding: Discovered the critical subdomain: web1337.inlanefreight.htb, which likely hosts a hidden administrative service.

Pivoted to FFUF for advanced directory brute-forcing.

ffuf -u http://web1337.inlanefreight.htb/FUZZ -w /usr/share/seclists/Discovery/Directories/common.txt

⚡ Exploitation

Deep crawling to extract sensitive data. Initial checks of robots.txt revealed un-indexed paths.

curl http://web1337.inlanefreight.htb:44038/robots.txt
⚠️ Important: The robots.txt file revealed a hidden directory: /admin/, explicitly excluded from search indexing.

Deployed ReconSpider to recursively crawl the web application structure and extract hidden files.

python3 ReconSpider.py http://dev.web1337.inlanefreight.htb:44038

🎯 Post-Exploitation

Reviewing crawled files and utilizing FFUF to discover hidden services and APIs.

✅ Success: Extracted API Key ([REDACTED]) from config.json and Email Address ([REDACTED]) from secret.txt.

Further API mapping confirmed a brute-forceable structure.

ffuf -u http://web1337.inlanefreight.htb/FUZZ -w /usr/share/seclists/Discovery/Web-Content/common-params.txt

📚 Key Learnings

  • Subdomain Enumeration is Critical: Tools like Gobuster and FFUF must be run with comprehensive wordlists and correct virtual host settings (--append-domain).
  • Check for Hidden Directories First: Always check robots.txt and sitemap.xml early, as they often reveal hidden paths (like /admin/) without requiring brute-forcing.
  • API Keys and Emails Are Gold: Sensitive data is frequently stored in simple files (config.json, secret.txt) within un-indexed directories.
  • Use Virtual Environments for Isolation: Running tools in a Docker container or isolated environment prevents system contamination and simplifies dependency management.
  • Common Pitfall: If Gobuster results are sparse, immediately switch wordlists or try passive DNS enumeration tools like dnsenum.

🛠️ Tools Used

  • WHOIS: Used to query domain registration details, providing the foundation for OSINT.
  • Gobuster: Employed for rapid subdomain and directory brute-forcing in vhost mode.
  • FFUF: Used for advanced, high-speed directory and parameter fuzzing.
  • ReconSpider: A full website crawling tool used to identify hidden files and accessible endpoints recursively.
  • Nmap: Used for port scanning and service version detection (e.g., Apache/2.4.41).
  • DirBuster: Used for legacy directory brute-forcing checks.
