Task: Discover Subdomains for inlanefreight.htb

Subdomains often hide critical assets like admin panels or misconfigured services. Before scanning, I ensured local DNS resolution was configured via the /etc/hosts file.

Next, I used **Gobuster** in virtual host mode to brute-force subdomains against the main domain using a comprehensive wordlist.

Key Finding: Discovered the critical subdomain: web1337.inlanefreight.htb, which likely hosts a hidden administrative service.

After identifying the key subdomain, I quickly pivoted to **FFUF** for advanced directory brute-forcing, leveraging its speed and flexibility.

üìö Key Learnings and Pitfalls

This module reinforced that web reconnaissance is a non-linear, multi-step process. Success relies heavily on combining the right tools and knowing how to troubleshoot common issues.

  • **Subdomain Enumeration is Critical:** Tools like Gobuster and FFUF must be run with comprehensive wordlists and correct virtual host settings (--append-domain).
  • **Check for Hidden Directories First:** Always check robots.txt and sitemap.xml early, as they often reveal hidden paths (like /admin/) without requiring brute-forcing.
  • **API Keys and Emails Are Gold:** Sensitive data is frequently stored in simple files (config.json, secret.txt) within un-indexed directories.
  • **Use Virtual Environments for Isolation:** Running tools in a **Docker container** or isolated environment prevents system contamination and simplifies dependency management.
  • **Common Pitfall (No Gobuster Results):** If initial results are sparse, immediately switch wordlists (e.g., to smaller, more targeted ones) or try passive DNS enumeration tools like **dnsenum**.
